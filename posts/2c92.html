C++ 多态性-虚函数

1、虚函数

在C++ 基类&派生类中讲过，基类对象 & 派生类对象具有赋值相容性

回顾：通过基类指针、基类引用，都只能访问到派生类 从基类中继承到的成员(基类子对象中的成员)，不能访问到派生类中定义的成员

如何通过基类指针pb、基类引用rb 访问派生类中定义(增加?)的成员？ 下面就来介绍一下：

1. 虚函数的定义与使用

  • 虚函数：用关键字virtual修饰的、类的成员函数；

    虚函数定义：语法形式

    class X{
    ……
    virtual  返回类型  函数名(函数参数表)   //虚(成员)函数声明
    ……
    }
  • 虚函数的意义（即虚函数的虚特征）
    基类指针（或引用）–>指向 派生类的对象时，通过该指针（或引用）访问派生类中的虚函数时，将调用该指针（或引用）实际所指对象的成员函数！

例子

class B{ public: virtual void f( ){ cout<<"B::f()";} }
class D:public B{ public: void f( ){cout<<"D::f()"; } }
void main(){
    D d;           //派生类对象d
    B * pb = &d;   //基类指针pb-->派生类的对象d
    pb->f( );      //输出结果是？
}
// 添加virtual    输出结果： D::f()
// 未添加virtual  输出结果： B::f()
  • 总结：通过指向派生类对象的基类指针（或基类引用）访问函数成员f()时，
  1. 非虚函数：由指针(或引用)的类型决定调用的版本；
  2. 虚函数：由指针(或引用)指向的实际对象决定调用的版本；

2.虚函数的使用方法：

步骤一：在基类中，用virtual声明成员函数为虚函数;

步骤二：在派生类中‘重定义’此函数;

  • 当一个成员函数被声明为虚函数后，其派生类（直接或间接）中原型相同的函数都将自动成为虚函数。

步骤三：定义一个指向派生类对象、基类类型的指针变量（或引用）;

步骤四： 通过该基类指针变量（或引用）调用此虚函数。

2、虚函数特性

  1. 一旦将某个成员函数声明为虚函数后, 如B::f()，它在继承体系中就永远成为虚函数了!
 #include <iostream>
using namespace std;
class A {
public: 
     void f(int i) { cout << "…A" << endl; };
};
class B : public A {
public:
     virtual void f(int i) { cout << "…B" << endl; }
};

class C : public B {
public:
     void f(int i) { cout << "…C" << endl; }
};
class D : public C {
public:    
     void f(int) { cout << "…D" << endl; }
};
int main() {
    A *pA, a;
    B *pB, b;    C c;    D d;
    pA = &a;    pA->f(1);    //调用A::f
    pA = &b;    pA->f(1);    //调用A::f
    pA = &c;    pA->f(1);    //调用A::f
    pA = &d;    pA->f(1);    //调用A::f
    return 0;
}  
/*输出：
        …A
        …A
        …A
        …A
*/
  1. 如果基类定义了虚函数，当通过基类指针（或引用）调用派生类对象时，将访问到它们实际所指对象中的虚函数版本。
#include <iostream>
using namespace std;
class A {
public: 
    virtual void f(int i) { cout << "...­A" << endl; };
};
class B : public A {
public:
     virtual void f(int i) { cout << "...­B" << endl; }
};

class C : public B {
public:
     void f(int i) { cout << "...­C" << endl; }
};
class D : public C {
public:    
     void f(int) { cout << "...­D" << endl; }
};
void main() {
    A *pA, a;
    B *pB, b;    C c;    D d;
    pB = &b;    pB->f(1);    //调用B::f
    pB = &c;    pB->f(1);    //调用C::f
    pB = &d;    pB->f(1);    //调用D::f
}
/*输出：
        …B
        …C
        …D
*/
  1. 只有通过基类对象的指针和引用访问派生类对象的虚函数时，才能体现虚函数的特性。
#include <iostream>
using namespace std;
class B{
public: virtual void f(){ cout<<"B::f()"<<endl; };
};
class D : public B{
public: void f(){ cout<<"D::f()"<<endl; };
};
void main(){
   D d;           //派生类对象d
   B *pB = &d; //基类指针pB ->指向：派生类对象d
   B &rB=d;        //基类引用rB ->指向：派生类对象d
   pB->f();        //调用D::f（体现虚函数特征）
   rB.f();        //调用D::f（体现虚函数特征）

   B b;            //基类对象b
   b=d;            //派生类对象d ->赋值给b(基类对象)
   b.f();        //调用B::f (“没有”体现虚函数特征)
}
/*输出：
        D::f()
        D::f()
        B::f()
*/
  1. 派生类中的虚函数要保持其虚特征，必须与基类中的函数原型完全相同，否则就是普通的重载函数，与基类的虚函数无关。

基类B和派生类D都具有成员函数f ，但它们的参数类型不同，因此不能体现函数f在派生类D中的虚函数特性。

#include <iostream>
using namespace std;
class B{
public: virtual void f(int i){ cout<<"B::f()"<<endl; };
};

class D : public B{
public: int f(char c){ cout<<"D::f()"<<c<<endl; }
};

void main(){
   D d;            //派生类对象d
   B *pB = &d; //基类指针pB ->指向：派生类对象d
   B &rB = d;  //基类引用rB ->指向：派生类对象d

   pB->f('6'); //调用B::f（“没有”体现虚函数特征）
   rB.f('B');    //调用B::f（“没有”体现虚函数特征）
}
/*输出：
        B::f()
        B::f()
*/
  1. 派生类通过从基类继承来的成员函数 调用 虚函数时，将访问到派生类中的版本。
    派生类D的对象 通过基类B的普通函数f()调用派生类D中的虚函数g()。
#include <iostream>
using namespace std;
class B{
public: 
    void f(){ g(); }    
    virtual void g(){ cout<<"B::g()"; }  //虚函数
};

class D : public B{
public: 
    void g(){ cout<<"D::g()"; }
};

void main(){
  D d;   //派生类对象d
  d.f(); //派生类对象d ->调用: 从基类继承的成员函数f() ->调用虚函数g()
}
//输出： D::g()

  1. 有3种类型的成员函数不能定义为虚函数

  2. 类的构造函数和静态成员函数不能定义为虚函数，只有类的非静态成员函数才能被定义为虚函数。

    • 原因：虚函数的虚特性只有在继承（层次）结构中才能够起作用，而构造函数、静态成员没有被继承。

  3. 内联函数也不能定义为虚函数。

    • 因为：内联函数采用的是静态联编的方式，而虚函数是在程序运行时才与具体函数动态绑定的，采用的是动态联编的方式；

    • 尽管虚函数在类体内进行定义，C++编译器也将它视为“非”内联函数；（类体内定义的成员函数 –> 默认被作为: 内联函数；）

#include <iostream>
using namespace std;
class B{
public:
    void f(){ cout<<"B::f()"<<endl; }
    virtual void vf(){ cout<<"B::vf()"<<endl;}
    void ff(){
        vf(); 
        f(); 
    }
    virtual void vff(){            //虚函数
        vf();
        f(); 
    }
};

class D: public B{
public:
    void f(){ cout<<"D::f()"<<endl; }
    void vf(){ cout<<"D::vf()"<<endl; }    
    void ff(){
        f(); 
        vf();
    }
};

void main(){
    D d;             //派生类对象d
    B * pB = &d;    //基类指针pB ->指向派生类对象d

    pB->f();        //?
    pB->ff();        //?
    pB->vf();        //?
    pB->vff();        //?
}

3、override和final （C++11）

1. override解决的问题：

class B {
     public: virtual void outData(int a) 
               { cout << a; };
};
class D : public B {
     public:    void  outData(double b) 
               { cout << b; }
};

派生类Ｄ本意
重定义从基类继承到的outData虚函数，但形参的类型不同。就不是重定义，编译器会为D生成两不同的重载版本的outData函数，没有实现虚函数的目的。

用override限定类D的outData函数，编译器就知道它是基类虚函数的覆盖函数版本。当该函数的参数表与基类的不同时，编译器就会检测到该错误，不允许通过编译。

class D : public B {
   public:void  outData(double b) override{ cout << b; }       //错误
};

2. override 注意事项：

  • override只能用在派生类中限定从基类继承到的虚函数。如果用它限定基类的非虚函数，或者派生类新定义的函数，则会产生编译错误。
class B {
public:
       virtual void g1(int a) { cout << a; };
       void g2(int b) { cout << b; }
};
class D : public B {
public:
      void  g1(int b)override { cout << b; }  //正确 
      void  g2(int b)override { cout << b; }  //错误,基类B的g2不是虚函数
      void  g3(inb b)override { cout << b; }  //错误，基类B没有g3函数
    void  g1(char b)override { cout << b; }  //错误，与基类的虚函数g1不匹配
};

4. 虚函数的实现技术

1. 虚函数的实现

  • 早期绑定与后期绑定

    • early-binding：编译时间完成的绑定

    • late-binding：运行时间完成的绑定

      虚函数是通过后期绑定实现的

  • 编译时行为(compiling-time)和运行时行为(running-time)

    • 编译时行为：通过静态分析就可以确定的行为

    • 运行时行为：通过动态分析（运行过程的分析）

      虚函数是通过动态行为分析实现的运行时行为

2. C++实现虚函数的技术

  1. C++通过虚函数表来实现虚函数的动态绑定。

    • 在编译带有虚函数的类时，C++将为该类建立一个虚函数表（vtable）；
    • 在虚函数表vtable中存放着：指向本类虚函数的指针，这些指针指向本类的虚函数地址；

  2. 在有虚函数的类对象中，C++除了为它保存每个数据成员外，还保存了一个指向本类虚函数表的指针（vptr）。

  3. 虚函数表vtable & 指向虚函数表的指针vptr
