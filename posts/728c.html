C++ 多态性-纯虚函数&抽象类

虚析构函数、纯虚函数与抽象类

1. 虚析构函数

  • 构造函数不能为虚函数
  • 析构函数可以定义为虚函数
  • 如果基类的析构函数是虚函数，则所有直接或间接从基类派生的类的析构函数都是虚函数；

  1. 使用虚析构函数的原因：在非虚析构函数的情况下，通过基类指针a 对 派生对象的析构可能是不彻底的。

     #include <iostream#&62
 using namespace std;
 class A{
 public:
     //virtual ~A(){ cout<<"调用A::~A()"<<endl; }  //声明为 “虚”析构函数

     ~A(){ cout<<"调用A::~A()"<<endl; }
 };
 class B:public A{
     char *buf;
 public:
     B(int i){    buf=new char[i]; }
     ~B(){
         delete [] buf;
         cout<<"调用B::~B()"<<endl;
     }
 };
 void main(){
     A* a=new B(10);    //基类指针a ->指向派生类对象new B(10)
     delete a;        //通过指针a ->删除 动态创建的 派生类对象new B(10)
 }
 /*  输出结果：
     调用A::~A()
     此结果表明: 没有析构buf

     声明为 “虚”析构函数 输出结果：
     调用B::~B()
     调用A::~A()
     此结果表明: 
     (1)先-调用派生类析构
     (2)再-调用基类析构

     回收了buf空间!!!
 */

  2. 基类析构函数几乎总是为虚析构函数。

    • 派生类的构造顺序: (1)先基类; (2)再对象成员; (3)后派生类本身；
    • 如果只调用基类析构函数，可能析构不彻底；

  3. 假定使用delete和一个指向派生类的基类指针来销毁派生类对象，如果基类析构函数不为虚，就如一个普通成员函数，delete函数调用的就是基类析构函数，这将致使对象析构不彻底！

2. 纯虚函数和抽象类

1. 纯虚函数

  • 概念：仅定义函数原型，而不定义其实现的虚函数；

  • 语法格式

      class X {
  virtual  ret_type   func_name (params) = 0;
  }

  • 需要纯虚函数的原因：

  1. 实用角度：占位手段（place-holder）

  2. 方法学：接口定义手段，抽象表达手段

2. 抽象类

  • 抽象类的定义：包含一个或多个纯虚函数的类；
  • C++对抽象类具有以下规定：

  1. 抽象类中必含有纯虚函数（至少1个纯虚函数）；

  2. 由于纯虚函数没有实现代码，所以不能建立抽象类的对象。即抽象类只能作为其它类的基类；

  3. 可以通过抽象类对象的指针（或引用）访问到它的派生类对象，实现运行时的多态性。

  4. 如果派生类只是简单地继承了抽象类的纯虚函数，而没有重新定义基类的纯虚函数，则派生类也是一个抽象类。

  5. 抽象类转化为实体类的方法是：定义一个抽象类的派生类，并定义所有纯虚函数！

  • 抽象类的主要用途——作为“接口”

#include <iostream#&62
using namespace std;
class Figure{                                    
protected:
    double x,y;
public:
    void set(double i,double j){ x=i; y=j; }
    virtual void area()=0;                                //纯虚函数
};

class Triangle:public Figure{
public:
    void area(){cout<<"三角形 面积："<<x*y*0.5<<endl;}    //重写基类纯虚函数
};

class Rectangle:public Figure{
public:
    void area(int i){cout<<"矩形 面积："<<x*y<<endl;}    //‘不是’基类纯虚函数的重写
}; 

int main(){
    Figure *pF;                //基类（抽象类）指针
    //  Figure f1;            //L1，错误 抽象类只能作为其它类的基类
    //  Rectangle r;        //L2，错误

    Triangle t;                //L3: 派生类（实体类）对象t
    t.set(10,20);

    pF=&t;                    
    pF->area();                //L4: 调用派生类的虚函数area()

    Figure &rF=t;
    rF.set(20,20);
    rF.area();                //L5
    return 0;
}
/*   输出结果：
    三角形 面积：100
    三角形 面积：200
*/

3. 运行时类型信息

  • 运行时类型信息（RTTI，Run-time Type Information）

    • 提供了在程序运行时刻确定对象类型的方法，是面向对象程序语言为解决多态问题而引入的一种语言特性；
    • 在最初的非多态程序程序设计语言中，并没有RTTI机制。

  • 在C++中，用于支持RTTI的运算符有：dynamic_cast，typeid。

  1. dynamic_cast的用途

    • dynamic_cast是一个强制类型转换操作符，主要用于多态基类的指针（或引用）与派生类指针（或引用）之间的转换，它是在程序运行时刻执行的。

    • 强制类型转换: const_cast、static_cast和reinterpret_cast, 则是在编译时期完成的。

  2. dynamic_cast的用法

     dynamic_cast(e)       //指针转换，e是指针
 dynamic_cast(e)     //引用转换，e必须是左值
 dynamic_cast(e)   //右值转换，e不能是左值
  • type必须是类类型，通常情况下type类型中应该有虚函数；
  • 当e和type之间是基类和派生类关系时，转换易成功。如果指针转换失败，dynamic_cast将返回0值；引用转换失败，dynamic_cast将会引发异常。
  1. dynamic_cast强制类型：

  • 向上转换是指在类的继承层次结构中，从派生类向基类方向的转换，即把派生类对象的指针（或引用）转换成基类对象的指针(或引用)，这种转换常用C++的默认方式完成。

  • 向下转换是指在类的继承层次结构中，从基类向派生类方向的转换，即把基类对象的指针或引用转换成派生类对象的指针或引用。

    向上转换 & 向下转换

      #include <iostream#&62
  using namespace std;
  class Base{
  public:
      virtual ~Base(){}
  };

  class Derived:public Base{
      void f(){ cout<<"f in Derived!\n"<赋值给:基类指针）默认转换，编译时完成，是常用方式

      pd=&d;        //派生类指针

      pb=dynamic_cast(&d);    //向上转换，运行时完成
      pb=dynamic_cast(pd);    //向上转换，运行时完成
  }

    向上转换 & 向下转换

      #include <iostream#&62
  using namespace std;
  class Base{
  public:
      virtual ~Base(){}
  };

  class Derived:public Base{
      void f(){ cout<<"f in Derived!\n"<(pb);    //L1：向下强制转换
      if(pd){
          cout<<"ok\n";   
      }else{
          cout<<"error!\n";
      }

      pd=dynamic_cast(&b);    //L2：向下强制转换
      if(pd){
          cout<<"ok\n";   
      }else{
          cout<<"error!\n";
      }

      pb=&d;    //（派生类对象的地址->赋值给: 基类指针）默认转换，编译时完成，是常用方式
      pd=dynamic_cast(pb);    //L3：向下强制转换
      if(pd){
          cout<<"ok!\n";   
      }else{
          cout<<"error!\n";;
      }
  }

  • 在默认情况下，在类的继承体系结构中，当用基类对象的指针操作（指向）派生类对象时，只能通过该指针访问派生类中的虚函数（这些虚函数也是基类中的虚函数）。

    那些在基类中没有被定义为虚函数，或派生类新增加的函数，通过基类指针是无法访问的。
