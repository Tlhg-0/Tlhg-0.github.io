基类&派生

一、基类&派生类的关系

1. 派生类拥有基类的数据成员和成员函数；

  • 派生类在基类的基础上可实施以下改变 ：

    1. 增加新的成员（数据+函数）。
    2. 重载(overload)基类的成员函数。
    3. 重定义(override,覆盖)基类已有的成员函数。
    4. 改变基类成员在派生类中的访问属性。

  • 派生类不能继承基类的以下内容：

    1. 析构函数。
    2. 基类的友元函数。
    3. 静态成员（数据+函数）
    4. 针对基类定义的一些特殊运算符，如new等。 基类的构造函数C++11之前不能继承，之后可以继承；

  • 基类的构造函数C++11之前不能继承，之后可以继承；

2. 派生类的访问权限

  • 派生类继承了基类的所有成员，但派生类能够直接访问从基类继承来的公有和保护成员，且只能通过这两类成员访问从基类继承来的私有成员。

    • 基类：
    class Base{
    base members;
};
    • 派生类：
    class Derived: public Base{
    new members;
};

  • 派生类对基类(子对象)成员的访问形式

    1. 通过派生类对象直接访问基类成员
    Derived d;
cout<<d.j;   d.fn2();
    1. 在派生类成员函数中直接访问基类成员 
    Derived d;
d.fn3();
    1. 通过基类名字限定访问被重载或重定义的基类成员名
    Derived d;
d.Base::fn1();
    • 有关派生类访问权限的例子：
    #include <iostream>
using namespace std;
class Base { //基类
public:
    int k;      //为演示,将数据成员声明为公有
    void fn1(){ cout<<"Base::fn1()"<<endl; }    
    void fn2(){ cout<<"Base::fn2()"<<endl; }
};
class Derived: public Base { //派生类
};
void main(){ Derived d;
    cout<<"k="<<d.k<<endl;
    d.fn1();
    d.fn2();
}

    结论： 派生类中没有新增加或者是重定义的成员，派生类Derived 拥有成员：k, fn1(), fn2()，派生类拥有的数据成员和基类是一样的。 

    #include <iostream>
using namespace std;
class Base { //基类
public:
    int k;      //为演示,将数据成员声明为公有
    void fn1(){ cout<<"Base::fn1()"<<endl; }    
    void fn2(){ cout<<"Base::fn2()"<<endl; }
};
class Derived: public Base { //派生类
public: int m;
    void fn3(){ cout<<"Derived::fn3()"<<endl; }
};
void main(){ Derived d;
    cout<<"k="<<d.k<<endl;
    d.fn1();
    d.fn2();
    cout<<"m="<<d.m<<endl;   d.fn3();
}

    结论： 派生类中有新增加的成员 m fn3()，派生类Derived 拥有成员：k, fn1(), fn2()，派生类拥有的数据成员和基类是一样的。

    (2) 重定义fn1()，隐藏了成员函数fn1()；

3. using与隐藏函数重现 (C++ 11)

1. 派生对基类同名成员的隐藏

  • 如果基类某个成员函数具有多个重载的函数版本，派生类又定义了同名成员，就会隐藏基类同名的全部重载函数。

2. 访问隐藏成员

  • 一是使用基类名称限定要访问的成员函数。

  • 重载基类的所有同名函数，而这些重载函数的代码与基类完全相同。

  • 用using声明使基类重载函数在派生类中可见。用法如下：using 基类名称::被隐藏成员函数名;

    例子：基类B的f1成员函数具有3个重载函数，派生D新增加了f1函数的功能，此f1会隐藏基类B中f1函数在派生类的可见性，用using将基类的f1引入到派生类作用域内

    #include <iostream>
using namespace std;
class B {
public:
  void f1(int a) { cout << a << endl; }
  void f1(int a,int b) { cout << a+b<< endl; }
  void f1() { cout << "B::f1" << endl; }
};
class D : public B {    
public:
  using B::f1;      //L1，使基类的3个f1函数在此区域可见
  void f1(char * d) { cout << d << endl; }
};
void main() {
  D d;
  d.f1();                           //L2，正确
  d.f1(3);                           //L3，正确
  d.f1(3, 5);                      //L4，正确
  d.f1("Hellow c++!");
  getchar();
}

4. 友元与继承

* 友元不能被继承

  • 每个类只能够负责控制自已的成员的访问权限。

  • 如果一个类继承了其它类，则它声明的友元也只能访问它自己的全体成员，包括它从基类继承到的public和protected成员。而它的基类和派生类并不认可这种友元关系，按照规则只能访问公有成员。

    例子：类Deri是基类Base的友元，函数f1和f2是类Deri的友元，分析下面程序中L4、L5、L7正确的原因，以及L8和L6错误的原因。

    #include <iostream>
using namespace std;
class Base {
public:
  int x = 0;
protected:
  double y = 0;
private:
  float z = 0;
  friend class Deri;                        //L1
};
class Deri :public Base {
protected:
  int dx = 1;
public:
  friend    void f1(Deri d);                 //L2
  friend  void f2(Base b);                 //L3
  void f3(Base b) { cout << b.x << b.y << b.z << endl; } //L4,正确
};
void f1(Deri d) {
  cout << d.x << d.y << d.dx << endl;   //L5，正确
//cout<<d.z<<endl;                 //L6，错误
}
void f2(Base b) {
  cout << b.x << endl;            //L7，正确
//cout << b.y << endl;            //L8, 错误
}
void main() {
  Base b;
  Deri d;
  f1(d);
  f2(b);
  getchar();
}

    解释：

    1. 由于类Deri是基类Base的友元类，所以类Deri中的成员函数可以访问Base中的所有成员，包括 public、protected、private 属性的。L4正确

    2. 由于类Deri继承了基类Base，则它声明的友元函数也只能访问它自己的全体成员，包括它从基类继承到的public和protected成员。不能访问基类的private成员。L5 正确 L6错误

    3. 由于类Deri继承了基类Base，基类Base并不认可类Deri与友元函数的关系，所以只能访问public成员。L7 正确 L8错误

5. 静态成员与继承

  • 基类静态成员为继承层次结构所有类共享；
  • 在继承体系中，如果基类定义了静成成员，则在整个继承体系中只有该成员的唯一定义，不论从该基类派生出了多少个或多少层次的派生类，静态成员都只有一个实例，为整个继承体系中的全体对象所共用。

6. 继承与类作用域

  1. 基类类域

    • 每个类都建立了属于自己的作用域，本类的全体成员都位于此作用域内，而且相互之间可以直接访问，不受定义先后次序的影响。
    • 例如，一个成员函数可以调用在它后面定义的另一个成员函数。

  2. 派生类类域

    • 派生类的作用域嵌套在基类作用域的内层。
    • 在解析类成员名称时，如果在本类的作用域内没有找到，编译器就会接着在外层的基类作用域内继续寻找该成员名称的定义。

  3. 编译器解析之后的派生类类域形式

     class A  {
   int g();……
 }；
 class B:public A{
   int h(int);……
 };
 class C:public B{
   int c；
   int h()；
   int f(int )；……
 }；

    经编译器处理之后，形成类似于下面的块作用域：

    A{
   int g(){……}
……
B { 
       int h(int ){……}；
       ……
       C {
          int c;
          int h(){……};
                   int f(int i){……;return  B::h(i);}    //L1
              ……
            }
    }
 }

 C xa;
 xa.g();
 xa.h(3);      //L2，错误
 xa.B::h(3);   //L3，正确

二、 基类对象 & 派生类对象的关系

1.基类对象 & 派生类对象赋值相容性

  1. 把派生类对象赋值给基类对象。

    • 把派生类中的‘基类子对象’的相应值 拷贝给基类对象；

  2. 把派生类对象的地址赋值给基类指针。

  3. 用派生类对象初始化基类对象的引用。

     Derived d;
 Base b=d;     
 Base *pb=&d;
 Base &rb=d;
  • 反之：则不行!!!

2. 把派生类对象赋值给基类对象——原理图

说明：不论以哪种方式把派生类对象赋值给基类对象，都只能访问到派生类对象中‘基类子对象那部份’的成员，不能访问派生类的自定义成员。

例子：

#include <iostream>
class A {
    int a;
public: 
    void setA(int x){ a=x; }
    int getA(){ return a;} 
}; 

class B: public A{
    int b;
public: 
    void setB(int x){ b=x; }
    int getB(){ return b;} 
};

void f1(A a, int x)   { a.setA(x); }
void f2(A *pA, int x) { pA->setA(x); }
void f3(A &rA, int x) { rA.setA(x); }

void main(){
    A a1, *pA;
    B b1;

    /*(0)*/
    a1.setA(1);
    b1.setA(2);
    //cou<<"a1.getA()=<<a1.getA(<<endl;        //a1.getA()=1
    //cou<<"b1.getA()=<<b1.getA(<<end<<endl;  //b1.getA()=2

    /*(1)派生类对象赋给基类对象*/
    a1=b1;
    cou<<"a1.getA()=<<a1.getA(<<endl;
    cou<<"b1.getA()=<<b1.getA(<<end<<endl;

    /*(2)*/
    a1.setA(10);
    cou<<"a1.getA()=<<a1.getA(<<endl;        
    cou<<"b1.getA()=<<b1.getA(<<end<<endl;    

    /*(3)派生类对象的地址赋值给基类指针*/
    pA=&b1; 
    pA->setA(20);
    cou<<"pA->getA()=<<pA->getA(<<endl;        
    cou<<"b1.getA()=<<b1.getA(<<end<<endl;

    /*(4)用派生类对象初始化基类对象的引用*/
    A &ra=b1;
    ra.setA(30);
    cou<<"ra.getA()=<<ra.getA(<<endl;    
    cou<<"b1.getA()=<<b1.getA(<<end<<endl;

    /*(5)*/
    b1.setA(8);
    cou<<"b1.getA()=<<b1.getA(<<endl;

    /*(6)值传递*/
    f1(b1,100); 
    cou<<"b1.getA()=<<b1.getA(<<endl;

    /*(7)地址传递*/
    f2(&b1,200);
    cou<<"b1.getA()=<<b1.getA(<<endl;

    /*(8)引用传递*/
    f3(b1,300);
    cou<<"b1.getA()=<<b1.getA(<<endl;
  }
  //运行结果：
  a1.getA()=2
  b1.getA()=2

  a1.getA()=10
  b1.getA()=2

  pA->getA()=20
  b1.getA()=20

  ra.getA()=30
  b1.getA()=30

  b1.getA()=8
  b1.getA()=8
  b1.getA()=200
  b1.getA()=300
